# Golden fixture contract — maze (`goldens/maze/`)

This file is the interface between the Python exporter (`tools/export_weights.py`
+ `tools/dump_goldens.py`, run in the upstream repo's uv env) and the Rust
consumers (`sheaf-io` loaders, `parity_check`, unit tests). Both sides implement
this document; neither side may drift from it silently. Goldens ARE committed
to git (they are small: B=2, K=12).

Shipped config: `configs/experiment/maze_sheaf.yaml` — MLPEncoderV2,
l1box_diag, LoRA rank 4 directional (8-way), ConcatMLPDecoderV2, prox-mode
unrolled CG T=5, d_v=10, d_e=5, gamma=5.0, rho_init=0.25, alpha=1.0,
z_init="h", prox_init="legacy", q_epsilon=1e-4.

Fixed sizes below assume the in-distribution 19×19 maze with patch_size=3,
stride=2, connectivity=8:

| symbol | value | meaning |
|---|---|---|
| B | 2 | batch (2 fixed mazes from the test set; if the dataset is unavailable, generated by `build_maze.py` at seed 0, first 2 accepted) |
| N | 81 | agents (9×9 grid of centers) |
| E | 272 | grid edges (8-conn, each undirected edge once, oriented right/down) |
| K | 12 | recorded ADMM iterations (`coordinate_history`, num_iters=12) |
| d_v | 10 | vertex stalk |
| d_e | 5 | edge stalk |
| r | 4 | LoRA rank |
| D | 8 | direction slots, order (N, NE, E, SE, S, SW, W, NW) = 0..7 |
| ps | 3 | patch size |
| C | 6 | token classes (0 empty, 1 wall, 2 start, 3 goal, 5 path) |

All floats are **f32**, all integers **i64**, everywhere (exporter casts; Rust
loaders reject anything else). Weights are dumped from the **EMA** tree unless
stated; the entire trace is generated with EMA weights (paper eval convention).

## Files

```
goldens/maze/
├─ config.json           # ExportedConfig (model + task + baked scalars)
├─ weights.safetensors   # params/... AND ema_params/... trees
├─ batch.npz             # the fixed input batch + derived views
├─ trace.npz             # every intermediate the Rust port must reproduce
└─ manifest.json         # exhaustive name -> {shape, dtype} for all of the above
```

## config.json

Three top-level objects; Rust deserializes with `deny_unknown_fields`
(`sheaf_nn::config::ExportedConfig`), so the exporter must emit exactly these
keys.

```json
{
  "model": {
    "num_classes": 6, "d_v": 10, "d_e": 5,
    "encoder_arch": "mlp_v2", "enc_hidden_dim": 256, "comm_norm_type": "layernorm",
    "objective_mode": "l1box_diag", "x_solver": "diagonal_prox",
    "z_solver": "unrolled_cg", "z_mode": "prox", "gamma": 5.0,
    "cg_iters": 5, "tikhonov_eps": 1e-5, "prox_init": "legacy",
    "rm_sharing": "directional", "rm_init": "orthonormal", "rm_mode": "context",
    "lora_rank": 4, "lora_alpha": 1.0, "lora_use_gate": false,
    "lora_init_style": "standard", "num_directions": 8,
    "relaxation_alpha": 1.0, "z_init": "h", "q_epsilon": 1e-4,
    "l1_init": 0.01, "upper_init": 1.0,
    "decoder_arch": "mlp_concat_v2", "dec_hidden_dim": 256
  },
  "task": {
    "task": "maze", "patch_size": 3, "stride": 2, "connectivity": 8,
    "num_classes": 6, "k_eval": 100, "loss_window": 4
  },
  "baked": {
    "rho": <float>
  }
}
```

`baked.rho` is `softplus(rho_raw + inverse_softplus(0.25))` evaluated at export
on the **EMA** `rho_raw` (PLAN.md §3.5: scalar reparameterizations are baked;
Rust inference never implements the offset-softplus). The raw `rho_raw` params
still appear in the safetensors for completeness but Rust ignores them.

## weights.safetensors — key naming

**Rule:** key = `<collection> "/" <flattened Flax module path>`, where
`<collection>` is `params` or `ema_params` and the Flax path joins the
`nn.compact` module names with `/` exactly as produced by
`jax.tree_util.tree_flatten_with_path`. No transposition, no reordering:
Dense kernels stay **[in, out]** (`y = x·W + b`).

`SheafADMMModel.init` on the maze config; 36 arrays, 181,859 params per
collection — the exporter asserts this list exactly, extra or missing keys are

## batch.npz

The fixed input batch and every graph/view tensor derived from it (dumped, not
recomputed, so Rust view code is cross-checked array-by-array):

| name | shape | dtype | producer |
|---|---|---|---|
| `tokens` | [B, 19, 19] | i64 | maze token grids (inputs) |
| `labels` | [B, 19, 19] | i64 | solved-maze token grids (ground truth) |
| `centers` | [N, 2] | i64 | `grid_agent_centers((19,19), 2, 3)`; (y, x), row-major |
| `node_positions` | [N, 2] | f32 | `centers` cast to f32 (model input) |
| `edges` | [E, 2] | i64 | `build_grid_edge_indices(centers, 2, 8)`; (u, v) |
| `dir_uv` | [E] | i64 | direction slot at u (`compute_direction_index(dy, dx, 8)`) |
| `dir_vu` | [E] | i64 | direction slot at v (delta negated) |
| `patches` | [N, B, 3, 3, 6] | f32 | `prepare_maze_patches(tokens, centers, 3, 6)` (wall pre-pad + one-hot) |

## trace.npz

Everything downstream of the weights, generated by
`model.apply(ema_params, patches, edges, num_iters=12, node_positions=...,
method=SheafADMMModel.coordinate_history)` plus the intermediate heads. K = 12.

Encoder heads (post-reshape to `[N, B, ...]`; `q_diag` includes the softplus
and the +1e-4 floor; `l1_weight`/`upper` are post-softplus):

| name | shape | dtype |
|---|---|---|
| `enc_h` | [N, B, 10] | f32 |
| `enc_q_diag` | [N, B, 10] | f32 |
| `enc_q` | [N, B, 10] | f32 |
| `enc_l1_weight` | [N, B, 10] | f32 |
| `enc_upper` | [N, B, 10] | f32 |
| `lora_A` | [N, B, 8, 5, 4] | f32 |
| `lora_B` | [N, B, 8, 10, 4] | f32 |

(No `lora_gate` array: `lora_use_gate=false`. `lower` is the hardcoded 0.)

Geometry (assembled from EMA base maps + slot tables):

| name | shape | dtype |
|---|---|---|
| `base_restriction_maps` | [E, 2, 5, 10] | f32 — slot 0 = F_{u→e}, 1 = F_{v→e} |
| `rho` | [] | f32 — must equal `config.baked.rho` bitwise |

ADMM trajectory (`ADMMHistory`, oldest first):

| name | shape | dtype |
|---|---|---|
| `x` | [K, N, B, 10] | f32 |
| `z` | [K, N, B, 10] | f32 |
| `y` | [K, N, B, 10] | f32 |
| `primal_res` | [K, N, B] | f32 — ‖x_relaxed − z‖₂ over d_v |
| `dual_res` | [K, N, B] | f32 — rho·‖z − z_prev‖₂ over d_v |
| `consistency` | [K, B] | f32 — sqrt(mean_{E,d_e} r² + 1e-6) |

Decoder outputs (per-iteration logits are decoded from the **x** iterates,
matching `coordinate_history`):

| name | shape | dtype |
|---|---|---|
| `logits_per_iter` | [K, N, B, 3, 3, 6] | f32 |
| `logits_final` | [N, B, 3, 3, 6] | f32 — `logits_per_iter[K-1]` (redundant on purpose: cheap first parity target) |
| `reassembled_final` | [B, 19, 19, 6] | f32 — overlap-mean reassembly of `logits_final` |

## manifest.json

One JSON object listing EVERY array the fixture set carries, so a Rust test can
verify completeness before any numeric comparison:

```json
{
  "config": "maze_sheaf",
  "generator": {"upstream_commit": "<sha>", "seed": 0, "B": 2, "K": 12,
                 "N": 81, "E": 272, "weights": "ema_params"},
  "safetensors": {
    "params/rho_raw": {"shape": [], "dtype": "F32"},
    "...": "... all 72 keys (36 per collection) ..."
  },
  "batch.npz":  { "tokens": {"shape": [2, 19, 19], "dtype": "i64"}, "...": "..." },
  "trace.npz":  { "x": {"shape": [12, 81, 2, 10], "dtype": "f32"}, "...": "..." }
}
```

Rust asserts: every manifest entry exists with the stated shape/dtype, and no
file contains an array absent from the manifest.

## Tolerances (consumer side, PLAN.md §5.2)

- weights / `base_restriction_maps` / `rho`: bitwise (pure loading + gather);
- encoder heads and `logits_final` at K=1: abs/rel ~1e-5;
- trajectory arrays: widening schedule starting ~1e-5 rel at iter 1 (f32 CG
  accumulation-order drift compounds); divergence is triaged with the f64
  build before being called a bug;
- `reassembled_final`: 1e-4 abs.
